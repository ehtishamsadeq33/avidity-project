In my existing React Native / Expo app, fix the top content being hidden behind the mobile status bar and header on all 4 report screens. Do exactly the following:
Root cause:

The content starts from y=0 but the mobile status bar and navigation header overlap it
Need to add proper safe area padding to all 4 screens

Fix for ALL 4 screens — ReportIntroScreen, ReportPage1, ReportPage2, ReportPage3:
Step 1 — Install safe area library if not already installed:
npx expo install react-native-safe-area-context
Step 2 — Wrap the entire app in SafeAreaProvider:

In app/_layout.tsx or App.tsx, wrap everything with:

javascriptimport { SafeAreaProvider } from 'react-native-safe-area-context';

export default function RootLayout() {
  return (
    <SafeAreaProvider>
      {/* existing app content */}
    </SafeAreaProvider>
  );
}
Step 3 — Fix ReportIntroScreen:

Import useSafeAreaInsets from react-native-safe-area-context
Add top padding to the ScrollView:

javascriptimport { useSafeAreaInsets } from 'react-native-safe-area-context';

const insets = useSafeAreaInsets();

// Change scroll style to:
<ScrollView
  style={[styles.scroll, { paddingTop: insets.top }]}
  contentContainerStyle={styles.scrollContent}
>

The black header should start BELOW the status bar not behind it

Step 4 — Fix ReportPage1:

Add useSafeAreaInsets hook
Add paddingTop: insets.top + 8 to the outermost View or ScrollView container
This ensures "Name:", "Gender:", "Age:" are visible without scrolling

Step 5 — Fix ReportPage2:

Same fix as ReportPage1
Add paddingTop: insets.top + 8 to the root View
"Name:", "Years in Organization" should be visible without scrolling

Step 6 — Fix ReportPage3:

Same fix
Add paddingTop: insets.top + 8 to the root View
"What do your results mean?" title should be visible without scrolling

Step 7 — Also fix bottom safe area:

On all 4 screens add paddingBottom: insets.bottom + 16 to the button row View at the bottom
This prevents the Back and Continue buttons from being hidden behind the home indicator on iPhone

Step 8 — Remove any hardcoded top margins or padding that conflict:

Search for any marginTop: 0 or paddingTop: 0 on root Views in these 4 screens and remove them
Search for any top: 0 absolute positioning that might override safe area

Do NOT change any report content, calculations, charts, PDF generation, MongoDB, navigation or survey screens
