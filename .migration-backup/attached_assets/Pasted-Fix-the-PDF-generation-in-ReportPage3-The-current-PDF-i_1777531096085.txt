🔹 PROBLEM 1: BARS/CHARTS NOT SHOWING IN PDF
═══════════════════════════════════════════════════════

expo-print uses a limited WebKit renderer that does NOT support:
- display: flex inside table cells or divs reliably
- percentage-width child divs inside flex containers
- inline-flex

Replace ALL bar charts in the HTML with TABLE-based bars instead.
This is the only reliable way to render progress bars in expo-print.

Use this exact pattern for every bar:

<table width="100%" cellpadding="0" cellspacing="0" style="margin:6px 0;">
  <tr>
    <td width="PERCENT%" style="background:COLOR;height:28px;border-radius:8px 0 0 8px;padding-left:8px;color:white;font-weight:bold;font-size:12px;">
      VALUE
    </td>
    <td width="REMAINDER%" style="background:#f0f0f0;height:28px;border-radius:0 8px 8px 0;">
    </td>
  </tr>
</table>

Where:
- PERCENT = the percentage value (e.g. 40)
- REMAINDER = 100 - PERCENT (e.g. 60)
- COLOR = the bar color
- VALUE = the label or score to show inside the bar

Apply this table-bar pattern to:
1. "What They Think" bar
2. "Where They Are" bar  
3. Every sub-bar inside Skills, Will, Environmental Support composites
4. The Directive vs Non-Directive bar

For the composite sub-bars (Skills/Will/Environmental), the bar 
width is (value / 10) * 100 percent since scores are out of 10.
